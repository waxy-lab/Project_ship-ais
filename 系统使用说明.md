## 1. 系统概述

本系统是一套基于 **ROS2** 的**水路仿真测试框架与智能避碰系统**，全称"Maritime Collision Avoidance System"。系统面向船舶航行安全领域，提供从场景仿真、碰撞风险评估、智能避碰决策到测试评估的完整闭环能力。

### 核心目标

- 模拟多种复杂船舶相遇场景（对遇、交叉、追越、包围、受限水域、恶劣天气等）
- 基于国际海上避碰规则（COLREGS）实现智能避碰算法
- 提供量化的算法性能评估与对比报告
- 支持实时可视化监控与事后数据回放
- 具备良好的扩展性，支持自定义算法插件

### 系统特点

| 特点 | 说明 |
|------|------|
| 模块化设计 | 6大独立包，低耦合高内聚 |
| ROS2原生 | 完整的话题/服务/消息定义 |
| 多场景支持 | 9种标准场景类型，10个预置配置 |
| COLREGS合规 | Rule 9/13/14/15/17/19完整实现 |
| 配置热加载 | 无需重启即可更新参数 |
| 算法插件化 | 支持动态加载自定义避碰算法 |
| 多格式导出 | JSON / CSV / TXT 报告输出 |

## 3. 模块详解

### 3.1 scenario_generator — 场景生成器

**路径：** `src/scenario_generator/`

负责生成各类标准化船舶相遇场景，输出 `ScenarioConfig` 对象或 YAML 配置文件。

| 文件 | 功能 |
|------|------|
| `generator.py` | 核心：`ScenarioGenerator` 类，包含所有场景生成方法 |
| `models.py` | 数据模型：`ShipState`、`ScenarioConfig`、`EnvironmentConfig` |
| `config_loader.py` | YAML配置加载与验证 |
| `scenario_loader.py` | 场景文件的读写（保存/加载YAML） |
| `ais_converter.py` | 将 `ScenarioConfig` 转换为 AIS 模拟器配置格式 |

**支持的场景类型：**

| 场景类型 | 方法名 | 参数类 |
|----------|--------|--------|
| 对遇 | `generate_head_on_scenario()` | `HeadOnParams` |
| 交叉相遇 | `generate_crossing_scenario()` | `CrossingParams` |
| 追越 | `generate_overtaking_scenario()` | `OvertakingParams` |
| 多船复杂 | `generate_multi_ship_scenario()` | `MultiShipParams` |
| 紧急避让 | `generate_emergency_scenario()` | `EmergencyParams` |
| 多船包围 | `generate_surrounding_scenario()` | `SurroundingParams` |
| 受限水域 | `generate_restricted_water_scenario()` | `RestrictedWaterParams` |
| 恶劣天气 | `generate_rough_weather_scenario()` | `RoughWeatherParams` |
| 能见度不良 | `generate_poor_visibility_scenario()` | `PoorVisibilityParams` |

### 3.2 collision_avoidance — 避碰决策模块

**路径：** `src/collision_avoidance/`

系统核心模块，集成风险评估、规则判定、路径规划、ROS2节点等完整避碰决策链路。

| 文件 | 功能 |
|------|------|
| `risk_assessment.py` | DCPA/TCPA/CRI计算，风险等级判定 |
| `rules_engine.py` | COLREGS规则引擎，相遇类型识别与避让动作输出 |
| `path_planning.py` | 候选路径生成、评估与最优路径选择 |
| `collision_avoidance_node.py` | ROS2节点，订阅AIS数据，发布决策和控制指令 |
| `visualization_publisher_node.py` | 实时发布可视化参数到ROS2话题 |
| `config_watcher.py` | 配置文件热加载监听器 |
| `algorithm_plugin.py` | 避碰算法抽象接口与插件注册表 |

### 3.3 ais_simulator — AIS模拟器

**路径：** `src/ais_simulator/`

模拟多艘船舶的实时运动，以1Hz频率输出标准AIS数据流，并响应外部控制指令。

| 文件 | 功能 |
|------|------|
| `ais_sim_node.py` | ROS2节点主体，船舶运动更新与AIS数据发布 |
| `map_loader.py` | 加载GeoJSON地图数据（水域边界） |
| `config/ships_config.yaml` | 船舶初始状态配置 |
| `config/river.geojson` | 默认水域地图 |
| `msg/AisShip.msg` | 单船AIS消息定义 |
| `msg/AisShipList.msg` | 多船AIS消息列表定义 |

**AIS数据字段说明：**

| 字段 | 含义 | 示例值 |
|------|------|--------|
| `message_type` | AIS消息类型（1=A类位置报告） | `1` |
| `mmsi` | 船舶唯一9位标识码 | `123456789` |
| `navigation_status` | 航行状态 | `Under way using engine` |
| `rate_of_turn` | 转向率（正=右转） | `53` |
| `speed_over_ground` | 对地航速（节） | `12.0` |
| `longitude` | WGS84经度 | `135.556325` |
| `latitude` | WGS84纬度 | `34.675717` |
| `course_over_ground` | 对地航向（度，0-360） | `124.3` |
| `true_heading` | 船首向（度，0-360） | `124` |
| `timestamp` | 当前分钟内的秒数 | `2` |
| `maneuver_indicator` | 机动标识（0=不可用/1=正常/2=特殊机动） | `Unknown` |

## 6. 场景生成

### 6.1 对遇场景 (Head-on)

两艘船相向航行，航向差接近180°。对应COLREGS Rule 14。

```python
from scenario_generator.generator import ScenarioGenerator, HeadOnParams

gen = ScenarioGenerator()
params = HeadOnParams(
    distance=3.0,        # 初始距离（海里）
    speed1=10.0,         # 本船速度（节）
    speed2=10.0,         # 目标船速度（节）
    base_latitude=30.0,  # 基准纬度
    base_longitude=120.0 # 基准经度
)
scenario = gen.generate_head_on_scenario(params)
```

### 6.2 交叉相遇场景 (Crossing)

两艘船以5°~112.5°的相对角度交叉航行。对应COLREGS Rule 15/17。

```python
from scenario_generator.generator import CrossingParams

params = CrossingParams(
    crossing_angle=45.0, # 交叉角度（度，5~112.5）
    distance=3.0,
    own_speed=10.0,
    target_speed=10.0
)
scenario = gen.generate_crossing_scenario(params)
```

### 6.3 追越场景 (Overtaking)

追越船在被追越船正后方22.5°扇形区域内，且速度更快。对应COLREGS Rule 13。

系统在追越完成后（本船超过目标船 > 0.1nm 且横向间距 > 0.3nm）自动发出 `RESUME_COURSE` 回正指令。

```python
from scenario_generator.generator import OvertakingParams

params = OvertakingParams(
    distance=0.8,
    own_speed=40.0,  # 追越船速度（必须 > target_speed）
    target_speed=12.0
)
scenario = gen.generate_overtaking_scenario(params)
```

### 6.4 多船包围场景 (Surrounding)

目标船被多艘船从四周包围，测试多威胁处理能力。

```python
from scenario_generator.generator import SurroundingParams

params = SurroundingParams(
    num_surrounding=4,        # 包围船数量（≥2）
    surrounding_distance=1.5, # 包围距离（海里）
    own_speed=15.0,
    min_speed=18.0,
    max_speed=22.0
)
scenario = gen.generate_surrounding_scenario(params)
```

### 6.5 受限水域场景 (Restricted Water)

在狭窄航道中的交叉场景，测试航道规则遵守。对应COLREGS Rule 9。

目标船从右前方225°斜向驶来，本船沿90°航道行驶，系统触发减速靠右避让而非大角度转向。

```python
from scenario_generator.generator import RestrictedWaterParams

params = RestrictedWaterParams(
    channel_width_nm=0.8,   # 航道宽度（海里）
    channel_length_nm=6.0,  # 航道长度（海里）
    channel_heading=90.0,   # 航道方向（度）
    own_speed=15.0,
    target_speed=12.0,
    separation_nm=1.0       # 初始间距
)
scenario = gen.generate_restricted_water_scenario(params)
```

### 6.6 恶劣天气场景 (Rough Weather)

在风浪流干扰环境中的相遇场景。系统自动对 `apply_colregs_rule` 传入环境参数，恶劣天气下额外增加 **10°** 转向余量（补偿风流漂移），同时减速至 **85%**，并在仿真步进中施加风流偏航扰动使船体轨迹不再是直线。

```python
from scenario_generator.generator import RoughWeatherParams

params = RoughWeatherParams(
    wind_speed_ms=15.0,      # 风速（m/s），>10m/s 触发恶劣天气模式
    wind_direction=45.0,     # 风向（度，0=北）
    current_speed_ms=1.5,    # 流速（m/s）
    current_direction=90.0,  # 流向（度）
    visibility_nm=3.0,       # 能见度（海里）
    distance_nm=1.2,         # 初始距离（海里）
    own_speed=20.0,
    target_speed=20.0
)
scenario = gen.generate_rough_weather_scenario(params)
```

**恶劣天气避让行为差异（对比基础对遇）：**

| 参数 | 基础对遇 | 恶劣天气 |
|------|---------|----------|
| 转向角度 | 15° | **25°**（+10°余量）|
| 避让速度 | 100% | **85%** |
| 船体偏航 | 无 | **有**（风流持续扰动）|
| 规则依据 | Rule 14 | Rule 14 + 天气修正 |

### 6.7 能见度不良场景 (Poor Visibility)

低能见度条件，系统自动采用保守风险阈值并按 **Rule 19** 操作。能见度不良时额外增加 **15°** 转向余量，减速至 **70%**，事件日志标注"Rule 19"。

```python
from scenario_generator.generator import PoorVisibilityParams

params = PoorVisibilityParams(
    visibility_nm=1.0,           # 能见度（海里），<2nm 触发 Rule 19
    distance_nm=1.2,             # 初始距离（海里）
    own_speed=15.0,
    target_speed=15.0,
    risk_threshold_warning=0.3,  # 预警阈值（低于正常值0.5）
    risk_threshold_danger=0.5    # 危险阈值（低于正常值0.7）
)
scenario = gen.generate_poor_visibility_scenario(params)
```

**能见度不良避让行为差异（对比基础对遇）：**

| 参数 | 基础对遇 | 能见度不良 |
|------|---------|------------|
| 转向角度 | 15° | **30°**（+15°余量）|
| 避让速度 | 100% | **70%** |
| 规则依据 | Rule 14 | **Rule 19** |
| 触发时机 | CRI≥0.5 | **CRI≥0.3**（更早预警）|

## 7. 碰撞风险评估

系统使用 **DCPA**（最近会遇距离）、**TCPA**（到达最近会遇点时间）、**CRI**（碰撞风险指数）三个核心指标评估碰撞风险。

### 7.1 基础用法

```python
from collision_avoidance.risk_assessment import assess_collision_risk

result = assess_collision_risk(own_ship, target_ship)
print(f"DCPA:    {result.dcpa:.2f} nm")
print(f"TCPA:    {result.tcpa:.2f} min")
print(f"CRI:     {result.cri:.3f}")
print(f"风险等级: {result.risk_level.value}")
```

### 7.2 风险等级说明

| 风险等级 | CRI范围 | 含义 |
|----------|---------|------|
| `SAFE` | < 0.6 | 安全，无需干预 |
| `WARNING` | 0.6 ~ 0.8 | 预警，密切关注 |
| `DANGER` | ≥ 0.8 | 危险，立即避让 |

> 能见度不良场景下阈值更低：WARNING阈值0.3，DANGER阈值0.5

### 7.3 多船风险评估

```python
# 对多艘目标船按风险排序，优先处理风险最高的
risks = [assess_collision_risk(own_ship, t) for t in target_ships]
risks.sort(key=lambda r: r.cri, reverse=True)
for r in risks:
    print(f"MMSI {r.target_mmsi}: CRI={r.cri:.3f} [{r.risk_level.value}]")
```

## 8. COLREGS规则引擎

### 8.1 相遇类型识别

```python
from collision_avoidance.rules_engine import determine_encounter_type, EncounterType

enc_type = determine_encounter_type(own_ship, target_ship)
# 返回值: HEAD_ON / CROSSING_GIVE_WAY / CROSSING_STAND_ON / OVERTAKING / SAFE

```

### 8.2 应用COLREGS规则

```python
from collision_avoidance.rules_engine import apply_colregs_rule

# 开阔水域标准避让（可选传入 environment 参数）
action = apply_colregs_rule(enc_type, own_ship, target_ship)

# 环境感知避让（恶劣天气/能见度不良自动调整角度和速度）
action = apply_colregs_rule(enc_type, own_ship, target_ship, environment=env_config)

# 受限水域专用规则（Rule 9）
from collision_avoidance.rules_engine import apply_restricted_water_rule
action = apply_restricted_water_rule(own_ship, target_ship)

print(f"避让动作: {action.action_type}")
print(f"转向方向: {action.turn_direction}")
print(f"转向角度: {action.turn_angle}°")
print(f"规则依据: {action.reason}")
```

### 8.3 COLREGS规则覆盖说明

| 规则 | 场景 | 系统行为 |
|------|------|----------|
| Rule 9 | 受限水域/狭水道 | 减速靠右行驶，不大角度转向（`apply_restricted_water_rule`）|
| Rule 13 | 追越 | 追越船右转绕行，超越完成后自动 `RESUME_COURSE` 回正 |
| Rule 14 | 对遇 | 双船均向右转向≥15°；恶劣天气+10°，能见度不良+15° |
| Rule 15 | 交叉（让路船） | 让路船右转≥15°；恶劣天气/雾中同步增加余量 |
| Rule 17 | 交叉（直航船） | 直航船保持航向航速，等待让路船行动 |
| Rule 19 | 能见度不良 | 减速至70%，转向角+15°，更早触发预警（CRI≥0.3）|

### 8.4 避让完成回正机制

系统在 `enc == NONE` 时自动检测避让是否完成，完成后返回 `RESUME_COURSE` 动作：

| 检测函数 | 适用场景 | 完成条件 |
|---------|---------|----------|
| `is_overtaking_complete(own, tgt)` | 追越 | 超前目标 >0.1nm 且横向间距 >0.3nm |
| `is_avoidance_complete(own, tgt)` | 对遇/交叉 | 目标在本船后方 >0.1nm 且间距 >0.5nm |

## 20. 常见问题与故障排除

**Q: 执行 `colcon build` 报找不到包？**
> 确认已安装ROS2并执行 `source /opt/ros/humble/setup.bash`，然后再构建。

**Q: AIS模拟器启动后无数据输出？**
> 检查 `ships_config.yaml` 中MMSI是否为9位数字，坐标是否在有效范围（纬度-90~90，经度-180~180），速度是否为正数。

**Q: 船舶位置不在预期水域？**
> 修改场景YAML中的 `base_latitude` 和 `base_longitude` 参数，使其与目标水域匹配。

**Q: 转换场景后AIS模拟器行为异常？**
> AIS模拟器当前不读取环境参数（风、流等），这些参数仅在 `collision_avoidance` 模块的风险评估中生效。

**Q: 配置热加载不生效？**
> 确认 `ConfigWatcher` 已设置 `auto_start=True`，且配置文件路径正确，文件有读取权限。

**Q: MQTT桥接无法连接？**
> 检查MQTT broker地址和端口配置，确认broker服务已启动，防火墙未拦截对应端口。

**Q: 运行测试时出现 `ModuleNotFoundError`？**
> 执行 `source install/setup.bash` 加载ROS2包路径，或在项目根目录执行 `pip install -e src/<package_name>`。

**Q: 如何调整风险预警阈值？**
> 修改 `config/system_config.yaml` 中的 `warning_threshold` 和 `danger_threshold` 字段，配置热加载会自动生效，无需重启。

## 21. 术语表

| 术语 | 全称 | 说明 |
|------|------|------|
| AIS | Automatic Identification System | 船舶自动识别系统，自动广播船舶位置、速度、航向等信息 |
| COLREGS | International Regulations for Preventing Collisions at Sea | 国际海上避碰规则，规范船舶相遇时的避让义务 |
